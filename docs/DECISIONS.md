- 결정: `docs/API.txt`를 단일 계약 문서로 사용
- 이유: 초기 구현 단계에서 API 변동 리스크 최소화

- 이유: API 명세에 명시된 범위 준수

- 결정: API 명세에 있으나 레퍼런스 이미지가 없는 화면은 신규 디자인으로 생성
- 이유: 레퍼런스가 전체 페이지를 커버하지 않으므로, 누락 화면은 기존 레퍼런스 스타일을 확장해 일관성 유지
