# Harness Runbook

## 1) 기능 시작 전

1. 명세 endpoint/에러코드 확인
2. 범위/제외 범위 확정
3. 테스트 시나리오 초안 작성

## 2) 구현 중

1. API 연동 단위별로 커밋/점검
2. 401 처리(refresh), 403/409 처리 분기 우선 검증
3. 실패 상태 UI를 마지막에 몰지 말고 함께 구현

## 3) 구현 후

1. happy path 점검
2. 에러 path 점검
3. 회귀 시나리오 점검
4. 문서 업데이트

## 4) 배포 전 체크

- 토큰/키/쿠키 노출 없음
- 인증 상태 파일 git 미포함
- 주요 플로우 수동 테스트 완료

## 5) 장애 대응

1. 에러코드/경로/재현절차 수집
2. 사용자 영향 범위 분류
3. 긴급 복구와 근본 원인 분리
4. 재발 방지 항목을 하네스 문서에 반영
