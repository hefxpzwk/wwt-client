# Agent Workflow

## 1) 티켓 생성

- 입력: 기능 요청 또는 버그 리포트
- 출력: 작업 목표, 범위, 제외 범위, 완료 조건

## 2) 설계 확정

1. API 명세 대응 endpoint 확정
2. 화면/상태 설계
3. 에러 코드 매핑 정책 확정

## 3) 구현

- API 클라이언트 -> 상태 관리 -> UI -> 에러 UX 순으로 구현
- 인증/권한 흐름은 우선 구현

## 4) 검증

1. happy path 통과
2. 주요 에러 path 통과(401/403/404/409/5xx)
3. 회귀 시나리오 통과

## 5) 인수인계

- 변경 파일 목록
- 테스트 결과
- 남은 리스크
- 후속 TODO

## 6) 커뮤니케이션 규칙

1. 애매한 요구사항은 작업 전에 명확히 정리
2. 추정이 필요한 경우 가정을 문서에 남기고 진행
3. 비즈니스 규칙 충돌 시 명세를 우선 적용
